Financial Data Audit (AAPL Stock)

Scenario: You have been provided with a raw CSV file containing Apple Inc. (AAPL) stock data. Before performing any financial modeling, your job is to perform an Initial Data Audit. You need to verify if the file loaded correctly, check the most recent records, and identify any missing values or structural errors.

1. What is Data?

In this case study, Data represents the historical movement of Apple's market value. Each "Observation" (row) represents a specific day, and each "Variable" (column) represents a specific metric like the price at opening or the total shares traded.

2. The Tool: Pandas Library

We use Pandas because it is designed specifically for tabular data. It allows us to load data into a DataFrame, which is a high-performance, two-dimensional data structure that makes "cleaning" much easier than using standard Python lists.

3. Loading and Reading Data

First, we import the library and read the file into our system memory.

In [1]:
import pandas as pd

# Reading the dataset
df = pd.read_csv('AAPL_stock_market.csv')

# Simply calling 'df' shows a preview of the entire table
print(df)
Explanation: The read_csv() function acts as a bridge, converting a flat text file into a functional table that Python can manipulate.

Data Cleaning Mastery

Amazon Sales Data Quality Audit & Business Preparation

What is Data Cleaning?

Data cleaning is the process of identifying and fixing problems in a dataset before analysis. It is about converting raw messy data into clean and reliable information.

🤔 Why is Data Cleaning Needed?

Real-world business data is rarely perfect. Errors occur in websites, billing systems, and forms. If businesses use "dirty" data, reports become inaccurate and decisions become risky.

The Gold Rule: Almost 80% of the analysis work happens during data cleaning. Machine learning and dashboards require accurate, clean input.

Organization Overview: Amazon

Amazon handles millions of orders daily. The analytics team stores data points for customers, products, payments, and ratings to drive strategic behavioral forecasting.

Business Context

The dataset contains critical variables: Order ID, Customer Name, Product, Sales, Profit, Rating, Payment Mode, and State.

Key Challenges

• Duplicate Records: Same order appearing multiple times.
• Missing Values: Incomplete sales or profit fields.
• Incorrect Data: Abnormal values due to system errors.

Project Objective

Clean and prepare the dataset to make it Structured, Reliable, and Analysis-Ready by removing noise and detecting abnormal outliers.

3. Checking Null Values

We check for "missing gaps" across the dataset before proceeding to imputation.

In [12]:
df.isnull().sum()
sales $: 1 | Profit: 1 | Customer_Name: 2

4. Sales Imputation (Mean)

Fills the gap in Order A102 with the average ($19,400).

In [13]:
df["sales $"] = df["sales $"].fillna(df["sales $"].mean())

5. Profit Imputation (Median)

Fills Arjun's missing profit with the median ($10,000).

In [14]:
df["Profit"] = df["Profit"].fillna(df["Profit"].median())

6. Standardizing Headers

Messy headers like "sales $" are converted to a professional format.

In [15]:
df.rename(columns={"sales $":"Sales_Amount"}, inplace=True)

7. Removing Outliers (Thresholding)

The $999,999 value for Keyboard is an obvious error. We remove it by keeping only orders under $60,000.

In [16]:
# Keeping only realistic sales values
df = df[df["Sales_Amount"] < 60000]
Simplified Thinking: Outlier detection protects your KPIs (like Average Ticket Size) from being skewed by single errors.
